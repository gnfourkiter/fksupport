Load Not Tracking - GPS

1. Check if the load is assigned with a truck/trailer number.

Here is an example from the shipper: Garushipper, have look at the load Test_for_deliveredAt. The load Test_for_deliveredAt is assigned with the truck 891 and the trailer 4189.

2. Get the truck/trailer number and run a ping test. We have two cases here:

• Case 1: If the truck/trailer number is not returning any locations while doing a ping test, then that is a GPS issue. Use the issue selector above to select the GPS issue and then click on 'Submit').

• Case 2: If the truck/trailer is returning locations (make sure that the timestamp is not older than 3 hours), and we aren't posting the locations on the platform, then that is a potential TRAC bug.

3. Now, we know that the locations aren't posting on to the platform. We need to continue to Logentries to research further.

Sometimes the load is assigned with the truck/trailer number, but we receive only the City and the State information. E.g. Danforth, IL. We have to double check if these locations are actually coming from the GPS location provider. Usually, the GPS locations have the distance and Zip codes included in the location information. E.g. 0.23 miles from Vernon, CA - 90058

4. Use the below search query in logentries to pull the locations received via GPS/ELD Provider.

<load_id> AND /Current location of load/ - in Global Worker logset. Look for errors in unique string.

5. Use the below search query in logentries to find if the checkcall creation had any errors.

CheckCallsController AND <load_id> - in Tracking Service logset. Look for errors in unique string.

6. Use the below search query in logentries to find errors while posting the location on to the platform.

/load_number/ AND POST_LOCATION_UPDATE - in Global Worker logset. Look for errors in unique string and don't forget to remove the trailing 0's from the load number.

7. Now, we know two things:

   a. This is not an issue with the GPS/ELD provider which we have already ruled out in step-2
   b. We have an issue where the locations aren't posting to the platform.

8. Regardless of error messages/codes, we have to create a TRAC bug because the issue is not in the integration, but with the platform.
